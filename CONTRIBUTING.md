# Flujo de contribuciones al repositorio

- Ver las issues que se te fueron asignadas en el sprint.  
- Escoge la issue que vas a resolver.  
- Crear una branch llamada `fix-issue-123` (reemplazar 123 con el número de la issue).  
- Dentro de esa branch hacer todos los cambios en el código necesarios para resolver esa issue.  
- Hacer commits y push de esta branch cuando esté lista.  
- Al menos un compañero distinto tiene que correr las pruebas para asegurar que todo funciona correctamente.  
  También aplicar un **linter** al archivo para asegurar que se está escribiendo con las convenciones correctas.  
- Se hará un **pull request** a producción y en la descripción se tiene que incluir:  
  `"Fixes #123"`.  
- El encargado de **DevOps** o **Project Manager** debe aceptar o bloquear el pull request.
